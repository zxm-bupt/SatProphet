### 1.3 发现的 Bug（测试过程中修复）

#### Bug-01：`TLERecord` 与 `SatelliteGroundTrack` 模型缺少 `__tablename__`

- **文件**: `app/models/tle.py`、`app/models/satellite_ground_track.py`
- **现象**: SQLModel 默认将类名转小写作为表名，导致查询 `tlerecord` / `satellitegroundtrack`，但迁移脚本创建的是 `tle_record` / `satellite_ground_track`，所有涉及 TLE 的 API 请求均返回 HTTP 500。
- **修复**: 在两个模型中均添加 `__tablename__` 显式声明。
- **影响**: 修复前 `/predict/{id}` 端点完全不可用（500 而非预期的 404/200）。

```python
# app/models/tle.py — 修复后
class TLERecord(SQLModel, table=True):
    __tablename__ = "tle_record"  # type: ignore[assignment]
    ...

# app/models/satellite_ground_track.py — 修复后
class SatelliteGroundTrack(SQLModel, table=True):
    __tablename__ = "satellite_ground_track"  # type: ignore[assignment]
    ...
```

## 三、错误日志分析

### Bug-01 关键日志（修复前）

```
sqlalchemy.exc.ProgrammingError: (psycopg.errors.UndefinedTable)
relation "tlerecord" does not exist
LINE 2: FROM tlerecord
             ^
[SQL: SELECT tlerecord.id, tlerecord.satellite_id, tlerecord.epoch, ...
FROM tlerecord
WHERE tlerecord.satellite_id = %(satellite_id_1)s::INTEGER ...]
```

**根因**: SQLModel 在没有显式 `__tablename__` 时将类名 `TLERecord` 直接转小写为 `tlerecord`，而 Alembic 迁移脚本显式创建的是 `tle_record`（带下划线）。两者不一致导致所有 TLE 查询失败，使预测接口始终返回 500 而非业务期望的 404/200。

**修复范围**: `app/models/tle.py`（+1 行）、`app/models/satellite_ground_track.py`（+1 行）。
