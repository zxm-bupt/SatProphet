# 项目自动部署与测试指令

你现在的任务是作为一个高级运维与测试工程师，根据本项目文档完成软件的自动化部署、功能测试，并生成最终报告。

## 1. 上下文注入
- **参考文档**: 优先阅读 `docs/` 目录下的安装指南、API 文档及部署手册。
- **项目结构**: 扫描根目录以识别技术栈（如 Go, Python, Node.js 等）。

## 2. 任务流水线

### 第一阶段：环境预检与部署
1. **依赖检查**: 检查当前环境是否满足文档要求的运行时版本（如 `go version`, `python --version`）。
2. **执行部署**:
   - 按照文档中的步骤执行安装命令。
   - 启动必要的本地服务或 Docker （podman）容器。
   - **验证**: 通过进程检查或端口扫描确认服务已成功拉起。

### 第二阶段：自动化测试
1. **编写测试代码**: 在项目根目录下创建 `claude-test/` 目录（如果不存在）。并在其中编写测试的代码。
2. **测试范围**: 执行`claude-test/` 中的集成测试。
3. **动态调试**: 如果部署或测试失败，请阅读错误日志，尝试修复环境或配置问题，并重新运行。
4. **性能/覆盖率**: 如果工具链支持，请收集代码覆盖率数据。

### 第三阶段：生成报告
1. **输出路径**: 在项目根目录下创建 `claude-test/` 目录（如果不存在）。
2. **文件命名**: 报告命名格式为 `claude-test/report-YYYYMMDD-HHMM.md`。
3. **内容要求**:
   - **部署摘要**: 记录部署步骤及结果。
   - **测试清单**: 列出所有测试用例及其状态（Pass/Fail）。
   - **错误日志**: 针对失败的任务，提取关键 Log 片段并给出初步分析。
   - **结论**: 给出“建议发布”或“存在阻塞性风险”的明确判断。

## 3. 约束与偏好
- **交互限制**: 尽可能自动尝试解决依赖冲突。
- **输出格式**: 测试报告必须使用 Markdown 格式。
- **图示规范**: 如果需要展示架构或流程，请使用 Mermaid 或 Plaintext（ASCII Art）。
